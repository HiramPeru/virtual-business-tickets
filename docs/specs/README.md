# Specs

Esta carpeta contiene las especificaciones activas e historicas del proyecto.

Cada spec describe una unidad de cambio: una mejora, correccion, modulo, migracion o decision de producto que requiere trazabilidad.

## Uso rapido

1. Copiar `TEMPLATE.md`.
2. Renombrar con `YYYY-MM-DD-slug.md`.
3. Completar las secciones necesarias antes de implementar.
4. Mantener el estado actualizado hasta `verified`, `deferred` o `cancelled`.
